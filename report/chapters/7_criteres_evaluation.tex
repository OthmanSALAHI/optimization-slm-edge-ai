\chapter{Evaluation Criteria}

\vspace*{0.3cm}

The optimizer comparison is evaluated using both quality metrics and efficiency
metrics. This is necessary because the project targets Edge AI applications,
where a model must be accurate but also fast, compact, and resource-efficient.

\section{Model Quality Metrics}

Validation loss measures the model error on data that was not used for parameter
updates. A lower validation loss indicates better generalization. Perplexity is
derived from the validation loss and is commonly used in language modeling:

\begin{equation}
    \text{Perplexity} = e^{\text{Validation Loss}}
\end{equation}

Lower perplexity means that the model is less uncertain when generating tokens.
In this project, validation loss and perplexity are used to compare the
fine-tuning quality obtained with each optimizer.

\section{Efficiency Metrics}

Training time and training speed indicate the computational cost of each
optimizer. Training time measures the total execution duration, while training
speed is expressed as the number of processed examples or tokens per second.

\vspace*{0.15cm}

Memory consumption is measured mainly through GPU memory usage. This metric is
important because Edge AI devices usually have limited memory. Model size is also
considered because smaller models are easier to store and deploy.

\vspace*{0.15cm}

Inference time and latency measure how fast the model generates outputs. Latency
is especially important for real-time applications, where the user expects quick
responses. Energy consumption is also relevant for Edge AI, but it depends on the
availability of reliable power telemetry.

\section{Compression Perspective}

The metrics collected during fine-tuning also provide a baseline for future
compression. A good model for compression should not only have low validation
loss, but also reasonable memory usage, latency, and model size.
