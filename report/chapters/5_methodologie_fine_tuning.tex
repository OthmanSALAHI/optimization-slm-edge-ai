\chapter{Fine-Tuning Methodology}

\vspace*{0.3cm}

This chapter describes the fine-tuning pipeline used to adapt
\texttt{google/flan-t5-small} to the prepared instruction-following dataset and
to compare several optimizers under the same conditions.

\section{Objective and Inputs}

The objective of fine-tuning is to update a pre-trained model so that it performs
better on a target task. In this project, the task is instruction following. The
model receives \texttt{source\_text} as input and learns to generate
\texttt{target\_text} as output.

\section{Model, Tokenizer, and Pipeline}

The model and tokenizer are loaded from Hugging Face. The tokenizer converts the
source and target texts into token IDs, with padding and truncation applied to
create fixed-size batches for GPU training.

\vspace*{0.15cm}

The training process is implemented mainly in \texttt{src/train.py}. It loads the
dataset, prepares the model and tokenizer, selects the optimizer, runs the
training loop, evaluates the model, and saves the outputs.

\section{Main Components}

\begin{itemize}
    \item \texttt{train.py}: controls the fine-tuning execution.
    \item \texttt{training\_config.py}: stores training parameters.
    \item \texttt{model\_utils.py}: loads and saves the model.
    \item \texttt{tokenizer\_utils.py}: prepares tokenized data.
    \item \texttt{optimizers.py}: selects the optimizer.
    \item \texttt{metrics.py}: computes benchmark metrics.
\end{itemize}

\section{Optimizer Commands}

The script accepts an optimizer argument, which makes it possible to compare
several methods with the same model and dataset.

\begin{lstlisting}[style=jsonStyle]
python src/train.py --optimizer sgd
python src/train.py --optimizer sgd_momentum
python src/train.py --optimizer adam
python src/train.py --optimizer lbfgs
\end{lstlisting}

\begin{figure}[H]
    \centering
    \includegraphics[width=0.55\textwidth]{chapters/images/optimizer_training_commands.png}
    \caption{Commands used to launch fine-tuning with each optimizer.}

All implemented optimizers can also be executed sequentially:

\begin{lstlisting}[style=jsonStyle]
python src/train.py --optimizer all
\end{lstlisting}

\section{Hardware and Outputs}

The fine-tuning process is designed to run on a CUDA-enabled GPU. In this project,
an RTX 5060 was used to execute the experiments. CPU execution is allowed only for
debugging:

\begin{lstlisting}[style=jsonStyle]
python src/train.py --optimizer adam --allow_cpu
\end{lstlisting}

Each optimizer saves a model under \texttt{results/models/} and a metrics file
under \texttt{results/metrics/}. This organization allows independent comparison
of training quality and efficiency.
